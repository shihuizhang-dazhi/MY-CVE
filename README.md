# MY-CVE

This repository stores CVE writeups and disclosure notes.

## How to use

- Add one Markdown file per CVE under `writeups/`.
- Recommended filename: `CVE-YYYY-XXXXX.md`.
- Include timeline, affected versions, root cause, impact, and fix.
- Keep PoC safe and for authorized testing only.

See `writeups/CVE-TEMPLATE.md`.
